# 输液瓶 PID 温控系统 — 需求与信息汇总  

> 版本: 2026-06-22-r2 | 基于原理图 `ProDoc_Schematic1_3_2026-06-22.epro2` + 用户补充

## 产品场景  

此模块用于加热盐水瓶给患者用于伤口的清洗。  
PI 加热片发热 → 外套布料保温 → 硅胶片接触热水瓶传导热量给盐水瓶。  
NTC 热敏电阻固定在盐水瓶两侧（被盐水瓶和硅胶片夹住）测温，**实际显示目标为瓶口温度**，需对 NTC 测量值进行硬编码补偿（类似额温枪补偿）。  

> 当前阶段目标：**先做 1 路加温测试，得到 PID 参数后再扩展 4 路。**

## 一、功能需求（已明确）  

### 1.1 温度控制  

| 项 | 要求 | 备注 |
|---|---|---|
| 调节范围 | 37℃ ~ 41℃ 可调 | 步进 **0.5℃** ✅ |
| 显示精度 | ≤ 0.1℃ | OLED 分辨率 |
| 控温精度 | 显示值与实际偏差 ≤ ±0.5℃ | ✅ PID 整定后达 **±0.3℃** |
| 默认温度 | **38℃** | ✅ 已定 (可通过 SW1/SW2 在 37-41℃ 调整) |
| NTC→瓶口补偿 | 硬编码 offset | ⏳ 默认 0, 实测标定中 |

### 1.1.1 ✅ 当前实测最佳 PID 参数 (2026-06-23 整定)  

> 在 1 路加热膜 (P7/D6/CH4) + 2 路 NTC (CH1=A0, CH2=A1, 取 max) 配置下的整定结果:

| 参数 | 取值 | 说明 |
|------|------|------|
| **Kp** (比例) | **35.0** | 适中, 保证追温推力 |
| **Ki** (积分) | **0.8** | **极小**, 防积分饱和导致冲过目标 |
| **Kd** (微分) | **25.0** | **强阻尼**, 对抗加热膜热惯性 |
| PID 输出上限 | **127** | = PWM 50% (`PWM_DUTY_MAX_PERCENT = 50`) |
| PWM 频率 | **~980Hz ≈ 1kHz** | D6 硬件 PWM (Timer0) |
| 采样周期 | **500ms** | dt ≈ 0.5s |
| 控温精度 | **37.5~37.7°C** (目标 38°C) | **误差 ±0.3℃, 完全达标** |
| 稳态 PWM 输出 | **~15%** (19/127) | 抵消散热维持温度 |

**整定历程**:
- 初值 (Kp=20, Ki=0.3, Kd=4) → 响应太慢
- 激进版 (Kp=50, Ki=5.0, Kd=12) → 超调冲到 39°C (超出 1°C), 回跌到 37°C
- **最终最佳** (Kp=35, Ki=0.8, Kd=25) → 无超调, 误差 ±0.3℃ ✅

**关键经验**: 该加热系统热惯性大,**D 项必须强**(25)、**I 项必须小**(0.8),否则积分饱和会推温度冲过目标。

### 1.2 安全保护（强制）  

1. **超温保护**: 实测温度 > 42℃ → 立即切断加热 + 蜂鸣报警 + OLED 显示报警  
2. **传感器故障保护**: NTC 短路或断路 → 自动停止加热 + 蜂鸣报警 + OLED 提示 (如 `ERR: SENSOR CH1`)  
3. **报警解除**: 故障排除后手动按键复位（不自动恢复）✅  

### 1.3 加热通道  

- 至少 2 路独立加温通道，预留 4 路  
- **原理图**确认：板上有 **4 颗 100N03 MOSFET**，对应 4 路 PWM 加热  
- ❓多通道独立控温还是同步？当前先做 1 路测试

- 连接器: **P3=CH1(A0), P2=CH2(A1), P8=CH3(A2), P9=CH4(A3)**  
- 当前只有 2 颗 NTC (P2 + P3)，预留 P8/P9  

### 3.2 分压电路 (Netlist 确认)  

```
VCC(5V)
  │
 10kΩ (R3=CH1, R4=CH2, R14=CH3, R15=CH4) — 0805 ±1%
  │
  ├──→ ADC (A0/A1/A2/A3)
  │
  │    100nF (C1~C4) ──→ GND (去耦)
  │
  ├──→ P3.2 / P2.2 / P8.2 / P9.2 ──→ NTC ──→ P*.1 ──→ GND
```  

> **接法 A**: VCC→10kΩ→ADC←NTC→GND  
> 温度↑ → NTC 阻值↓ → ADC 电压↓  
> **短路故障**: NTC=0Ω → ADC≈0  
> **断路故障**: NTC=∞Ω → ADC≈1023  

### 3.3 温度转换公式  

```cpp  
// 由 ADC 反算 NTC 电阻  
float voltage = (float)analogRead(PIN) * 5.0 / 1023.0;  
float R_ntc = 10000.0 * voltage / (5.0 - voltage);  
// 或用整数: R_ntc = 10000.0 * adc / (1023.0 - adc);  

// B 值公式 (简化 Steinhart-Hart)  
float steinhart = log(R_ntc / 10000.0);          // R25 = 10kΩ  
float tempK = 1.0 / (1.0/298.15 + steinhart/3950.0);  
float tempC = tempK - 273.15;  

// 瓶口温度补偿  
float displayTemp = tempC + OFFSET_BOTTLE;       // OFFSET 待标定  
```  

### 3.4 故障检测  

```cpp  
#define ADC_SHORT_THRESHOLD  20    // < 20 ≈ NTC < 200Ω → 短路  
#define ADC_OPEN_THRESHOLD   1000  // > 1000 ≈ NTC > 400kΩ → 断路  

bool sensorFault(int adc) {  
    return (adc < ADC_SHORT_THRESHOLD || adc > ADC_OPEN_THRESHOLD);  
}  
```  

| 状态 | ADC 值 | R_ntc | 温度 |  
|---|---|---|---|  
| 正常 25℃ | ~511 | ~10kΩ | 25.0℃ |  
| 正常 38℃ | ~403 | ~6.5kΩ | 38.0℃ |  
| 正常 42℃ | ~327 | ~4.7kΩ | 42.0℃ |  
| **短路** | 0~20 | <200Ω | → ERR |  
| **断路** | 1000~1023 | >400kΩ | → ERR |

## 四、PID 算法设计  

### 4.1 控制框架  
- **单向加热**（无制冷），PWM duty 0~255 经 MOSFET 驱动加热片  
- **PID 采样周期**: 500ms  
- **PWM 周期**: 1~2 秒（热惯性大，用慢速软 PWM）  

### 4.2 PID 结构（每通道独立实例）  

```cpp
struct PIDChannel {
    float Kp, Ki, Kd;
    float setpoint;       // 目标温度 (℃)
    float integral;       // I 累加器
    float lastError;      // 上次误差
    float output;         // 0~255 PWM duty
    bool  enabled;        // 该通道使能
    bool  alarm;          // 报警态
};

float computePID(PIDChannel &ch, float temp, float dt) {
    float error = ch.setpoint - temp;
    ch.integral += ch.Ki * error * dt;
    ch.integral = constrain(ch.integral, 0, 255);  // 抗饱和
    float derivative = (error - ch.lastError) / dt;
    ch.lastError = error;
    float out = ch.Kp * error + ch.integral + ch.Kd * derivative;
    ch.output = constrain(out, 0, 255);
    return ch.output;
}
```  

### 4.3 安全层（叠加到 PID 输出之上）  

| 条件 | 动作 |
|---|---|
| temp ≥ 42℃ | 强制 output=0, 置 alarm, 蜂鸣 |
| sensor fault | 强制 output=0, 置 alarm, 蜂鸣 |
| 开机前 30s | 限制 output ≤ 128 (50%) 防冲温 |

### 4.4 整定  
- 经验初值: **Kp=20, Ki=0.3, Kd=4**（1 路测试时通过 Serial Plotter 观察调优）  
- 测试阶段通过串口打印 `setpoint, temp, output` 三项，用 Arduino IDE Serial Plotter 观曲线整定

## 九、⚠️ 方案重大变更 (2026-06-30)

### 9.1 新方案概要

**原方案已废弃**: PI 加热膜包裹盐水瓶 → 加热速度过慢, 不满足使用需求。

**新方案**: 盐水经**加热管道**流动加热。
- 隔膜泵驱动盐水从塑料瓶输出, 流过加热管道时被加热
- 流速: 1~4 L/min (测试用最低 1 L/min)

# 新加入的状态
这班代码在外面进行一系列的测试后发现PI片的模式存在问题，即加热速度过慢。

·加温条长度：1400mm±50mm
